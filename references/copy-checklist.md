# 文案自检闸（出稿前逐条过，违反即改）

> 这是 skill 的**命门**。AI 一次写"过关"不现实——领导抠得细。本清单是**机械可判的黑名单**：拦住写得烂，比追求写得好更可靠。
> **出 `.card` 前 + 发预览前各过一遍。** 每条都是二元判定（违没违一眼看出）。

## 🚫 雷区黑名单（硬 DON'T）

1. **加粗关键词不带逗号/标点**，且同组**字数尽量一致**。
   - ❌ `**一键关联，零开发**`（逗号、6字） ✅ `**零开发接入**`/`**应用内直达**`/`**自动懂场景**`（5字、无标点）
2. **同组 bullet 描述长度接近**（默认宽度下"要么都不换行、要么都换行"）。长短不一→先拉齐，否则一条单独换行就乱。
3. **不用专有名词 / 英文缩写 / 黑话**：`SOP`、`API`、`PV/UV`、`运行态/配置态`。
   - 换词**不是逐字直译**：`SOP→问操作` 这种硬翻一样没人懂。替换词必须"没背景的人一眼懂"；找不到又准又白的短词，就**换一个具体例子/概念**（`SOP→看考勤` 这种）。
   - 例外：`CRM` 这类已被广泛认知的品类名。
4. **忌绝对化 / 夸大词**：`零`、`100%`、`完全`、`彻底`、`秒`。改用留余地的 `低`、`快`、`更`、`显著`。（领导对此最敏感）
5. **不口语/俚语**：`拿个`、`建个`、`选个`、`绑上去`、`怎么搞`、`怎么 X`。用朴实书面词：`添加`、`配置`、`查看`、`打开应用时`。
6. **少用「用户 / 你」主语，直接动宾**：corpus 验证 114 条亮点仅 5 条含"用户"。写 `识别迟到早退异常`，不写"用户可以识别…"。
7. **加粗只给关键词**，不整句加粗；冒号在加粗**外**（`**关键词**：描述`，不是 `**关键词：**描述`）。
8. **「告别 X」每张卡 ≤ 1 处**（品牌级修辞，滥用即廉价）。
9. **emoji 总数 ≤ 8**（含飞书 shortcode）；`🔥` 必双出 `🔥🔥`，不单用。
10. **冒号用中文 `：`**，不用英文 `:`。

## ✅ 正向导向（DO，让首稿起点不跑偏）

- **卖点讲"收益"不讲"实现手段"**：`零开发接入`（手段）→ `构建成本低`（收益，正中搭建者关切）。
- **动词具体落到真实场景**：`查库存`、`报线索`、`看考勤`，别用"办事/找指引"这类抽象词。
- **量化优先**：有数字用数字（`200 万行`、`100 个`），没有就用比喻，别说"很多/提升"。
- **痛点用提问式**：`订单总延期？` 紧接解法（合集模板模块尤其）。
- **称呼语统一**：默认「你」；正式/B 端用「您」；一张卡不混用。
- **标题结尾 emoji 看气氛**：`🎉`（能力上新）/ `🔥🔥`（重磅·邀你体验）。纯功能可不带。
- **AI 卡 slogan 用 `:GoGoGo:` 前缀**（飞书 shortcode，仅飞书内渲染）。

## 🙋 出稿时主动标不确定点

不要全篇当定稿丢给用户扫雷。**自己拿不准的地方主动标出来**，把用户注意力导到该抠的句子：
> "这个卖点我吃不准够不够具体，你重点看这句：……"

人工微调是流程的一部分，不是失败。用户每纠一次 → 按 [lessons.md](lessons.md) 沉淀回规则，下次更准。
